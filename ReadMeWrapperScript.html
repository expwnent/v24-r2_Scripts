Overview

The wrapper script allows for more customization when calling scripts. The game currently only allows very limited requirements when specifying targets for syndromes and interactions. This script provides dozens of new targeting options.

Requirements

To use the Event System you will need

  1. Dwarf Fortress
  2. DFHack
  3. Roses Script Collection

Options

There are currently over 30 different options available to use. They can be broken down into four main categories. Number Based, Token Based, Target Manipulation, and Script Manipulation.

Number Based Options

The number based options are

  • -age
  • -speed
  • -physical
  • -mental
  • -skills
  • -traits

Each of these options has the requirements min, max, greater, less. Min and max are straightforward, they tell the script that for a target to be acceptable they must have a minimum or maximum amount of a certain type.

Greater and less perform slightly differently. They take the ratio unitSource/unitTarget and compare it to a given value. The options can further be broken down into two subcategories, Subtype Based and Non-Subtype Based

Subtype Based Number Options

These options also require a further token to function

  • -physical - check the units physical attributes
  • -mental - check the units mental attributes
  • -skills - check the units skills
  • -traits - check the units traits

Non-Subtype Based Number Options

These options do not require a futher token to function

  • -age - checks the units age
  • -speed - checks the units speed

Example Number Based Options

To require a target unit has at least 2000 strength

-physical STRENGTH:min:2000

To require the target unit has willpower twice that of the source unit

-mental WILLPOWER:greater:2

To require the target unit be less than 100 years old

-age max:100

Token Based Options

The token based options are

  • -aclass
  • -acreature
  • -asyndrome
  • -atoken
  • -iclass
  • -icreature
  • -isyndrome
  • -itoken
  • -noble
  • -profession
  • -entity

These are also broken down into two subcategories, those requiring an additional option, and those that don’t.

Subtype Based Token Options

These options require an additional token to function

  • -noble - checks the units noble position
  • -profession - checks the units profession
  • -entity - checks the units entity

Either ‘required’ or ‘immune’ is needed following these options in order to function.

Non-Subtype Based Token Options

These options do not require an additional token to function

  • -aclass, iclass - checks the units [CREATURE_CLASS]
  • -acreature, icreature - checks the actual creature:case combination
  • -asyndrome, isyndrome - checks if the unit has the specified [SYN_CLASS]
  • -atoken, itoken - checks if the unit has the specified token

For these options the ‘a’/’i’ versions work like the ‘required’/’immune’ options do above.

Example Token Based Options

Only targets units that are the monarch noble

-noble required:MONARCH

Only targets dwarfs

-acreature DWARF:ALL

All creatures with the FLIER token are immune from targeting

-itoken FLIER

Target Manipulation Options

Now we get into the more advanced options. These options are

  • -radius
  • -plan
  • -maxtargets
  • -target
  • -reflect
  • -silence

Each one is fairly different, so lets take them one at a time.

-radius allows for selecting multiple targets in a given area around the -unitTarget. This means that you could have all units within 10 tiles, all units in a single square, or all units in a line be effected. The default value for this is -1,-1,-1 which means just the -unitTarget is effected.

-plan functions similarly to -radius but allows for more interesting shapes. It requires an external text file, located in the hack/scripts folder, more information on -plan can be found in an addendum

-maxtargets is useable with -radius and -plan to limit the number of targets that can be selected. If no -maxtargets option is selected, all the targets found will be targeted, otherwise a random sample, the size of -maxtargets will be selected from the list of available targets.

-target allows configurable targeting based on the relationship between the -unitSource and -unitTarget. The valid options for -target are

  • invasion
  • civ
  • population
  • race
  • sex
  • caste
  • enemy

-reflect and -silence both take [CREATURE_CLASS] and [SYN_CLASS] tokens as their arguments, but check differently.

-reflect checks the -unitTarget's creature classes and any active syndrome classes, and if any are found to match the given token the -unitTarget will be changed to be the -unitSource, and as long as the -unitSource passes the other options, the script will treat the -unitSource as the -unitTarget and the -unitTarget as the -unitSource.

-silence checks the -unitSource's creature classes and any active syndrome classes, and if any are found to match the given token the interaction simple won't be cast, effectively "silencing" the unit.

Example Target Manipulation Options

To target all units in a 5x5 square

-radius 5,5,0

To target only 2 of the units in the 5x5 square

-radius 5,5,0 -maxtargets 2

To target only units of the same race as the user

-target race

Script Manipulation Options

The final group of options are, possibly, the most unique.

  • -chain
  • -center
  • -delay
  • -value

-chain allows for "chaining" of spells. By default spells do not chain (i.e. -chain 0), but say you want a steel bolt to hit the first target and then hit another target (from the acceptable target list) you would use.

-chain 1

Note that this means you MUST HAVE a -radius or -plan option specified, otherwise it will just continually hit the same target (as there is no one else to chain to). Also note that by default this example

-radius 5,5,5 -chain 1

Will hit all units within a block of 5x5x5 around the target AND then each one of those will chain to another target (thus if there are 5 eligible units it will target all 5, then each of those 5 will chain to a new set of units within 5x5x5 of them). To change this behavior so that it still checks the targets in range, but only actually hits the -unitTarget, you must use the -center option, so that

-radius 5,5,5 -chain 1 -center

Would only hit the -unitTarget and then select a random unit from those otherwise eligible to hit next.

-center forces the script to ignore any previous options and only target the -unitTarget (note that it still keeps a list of otherwise illegible targets for use in other options, like -chain and -value).

-delay simply delays the effect of the script by a specified amount of in-game ticks.

-value, this is personally my favorite option, and probably the most complex. It effectively allows you to pass different arguments to scripts based on the units targeted and unit using the interaction. My go-to example is if you want your warrior to have a "battle shout" type ability that gives their willpower to all nearby friendly units you can do it with this option.
